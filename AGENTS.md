# Curio Landing Instructions

## Project context
- This repo is the public landing page for Curio.
- The landing is a static, Spanish-language acquisition page for Latin America.
- Its job is to explain Curio quickly, rank for search, and push users into the app.

## Product rules
- Preserve the editorial, warm Curio visual language.
- Keep the copy direct and minimal.
- Optimize for conversion and SEO, not for feature density.
- Do not turn the landing into a generic SaaS page.

## SEO and content rules
- Keep the main keyword focus on Spanish curiosity and discovery intent.
- Prefer headings and copy that reflect how people actually search in Spanish.
- Use FAQ, schema, and structured on-page sections when they help search and clarity.
- Keep the landing aligned with the content plan and social channels.

## Git and verification
- Do not use destructive Git commands.
- Make narrow commits for landing changes.
- Verify the page after edits with a build or local preview when possible.

## Useful references
- `CONTENT_PLAN.md` in the Curio workspace for acquisition strategy.
- `BRIEF_CURIO_CLAUDE_DESIGN.md` for brand and mascot direction.
